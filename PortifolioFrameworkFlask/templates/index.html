CLASS 04 (10/30)

Creating A New Virtual Environment And Developing Code

Always import Flask when creating a new virtual environment: from flask import Flask

Key Concepts:

  • Every web request works with RR (Request and Response).
  • The web environment operates reactively‚Äîit only acts when a request is made.
  • Flask allows the Python code to listen for incoming requests and respond accordingly.
  • First, the request reaches Flask, then, Python processes the request and returns a response.

The first step is to create the Flask application.

The application variable can have any name, but it is commonly named app.py

To automatically updates the server on code changes: app.run(debug=True)

To change the default port: app.run(debug=True, port=XXXX)

To make the application available to everyone on the same network: app.run(debug=True, port=XXXX, host='0.0.0.0')

For a web application to be reactive, it needs routes. Routes must be linked to a function that defines the response.

This is how to create different web pages:

@app.route('/contact')
def contact():
return 'This is the user page'

CLASS 06 (11/13)

Basic Rules For Development

1st Do not trust the data coming from the request

  • Validate all data; this validation depends on what is expected for each data type (e.g., text, number, etc.);

2nd Ensure data integrity and consistency

  • Ensure that what you received will be useful to calculate what is needed
  • Store it so it can be recovered if necessary;
  • We use persistent storage to preserve this data because persistence ensures durability and reliability;

3rd Work either asynchronously or synchronously

  • If synchronous, you must return the response in a timely manner;
  • In web development, synchronous methods are usually used;

4th Handle exceptions

  • Do not let the backend crash;

NOTE: In Python, we validate things through selection structures (what we know as conditional structures, but we cannot call them conditionals because these structures are used for selection purposes).

Validate the data, handle exceptions, and then persist it in database systems.

We will use SQLite
